# Test Cases

## TC-001: Получение списка пользователей

**Шаги:**

1. Отправить GET запрос `/users?page=2`

**Ожидаемый результат:**

* Статус код: 200
* В ответе присутствует список пользователей
* Ответ в формате JSON

---

## TC-002: Получение одного пользователя

**Шаги:**

1. Отправить GET запрос `/users/2`

**Ожидаемый результат:**

* Статус код: 200
* В ответе данные пользователя
* Поля id, email, first_name, last_name заполнены

---

## TC-003: Создание пользователя

**Шаги:**

1. Отправить POST запрос `/users`
2. Передать body:

```
{
  "name": "Tamila",
  "job": "QA Engineer"
}
```

**Ожидаемый результат:**

* Статус код: 201
* Пользователь создан
* В ответе есть id и createdAt

---

## TC-004: Обновление пользователя

**Шаги:**

1. Отправить PUT запрос `/users/2`
2. Передать обновленные данные

**Ожидаемый результат:**

* Статус код: 200
* Данные пользователя обновлены
* В ответе есть updatedAt

---

## TC-005: Удаление пользователя

**Шаги:**

1. Отправить DELETE запрос `/users/2`

**Ожидаемый результат:**

* Статус код: 204
* Пользователь удален

---

## TC-006: Создание пользователя без данных (негативный тест)

**Шаги:**

1. Отправить POST `/users` без body

**Ожидаемый результат:**

* Статус код: 400 (ожидаемо)
* Сообщение об ошибке
